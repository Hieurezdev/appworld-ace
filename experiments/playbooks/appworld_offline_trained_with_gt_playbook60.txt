[shr-00022] Always verify the correct data source when the task mentions 'album library' vs 'song library'

[shr-00024] Always check for existing reviews before creating new ones
- Before calling `review_song` or any API that creates a new resource, first use `show_song_reviews` to verify if a review already exists for the song. If a review exists and its rating is less than 5.0, create a new review with rating 5.0 to replace it. If no review exists, create one with rating 5.0. This prevents 409 Conflict errors and ensures the task is completed correctly without violating API constraints.
[shr-00025] Always adjust rating thresholds to match task-specific requirements
- When a task specifies a particular rating (e.g., 4-star), adjust the review update threshold accordingly. This prevents 409 Conflict errors and ensures the task is completed correctly.
