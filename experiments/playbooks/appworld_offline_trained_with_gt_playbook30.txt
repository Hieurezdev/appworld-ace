[shr-00022] Always verify the correct data source when the task mentions 'album library' vs 'song library'

- Before calling `review_song` or any API that creates a new resource, first use `show_song_reviews` to verify if a review already exists for the song. This prevents 409 Conflict errors and ensures the task is completed correctly without violating API constraints.
[shr-00025] Always adjust rating thresholds to match task-specific requirements
- When a task specifies a particular rating (e.g., 4-star), adjust the review update threshold accordingly. This prevents 409 Conflict errors and ensures the task is completed correctly.
